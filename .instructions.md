# Project Instructions for week2-lab2

## Project Overview

**week2-lab2** is an educational assignment management portal for Mergington High School's Computer Science course. It displays Python programming assignments with automatic due-date tracking and provides dynamically-rendered assignment details.

- **Tech Stack**: HTML5, vanilla JavaScript (ES6+), CSS3 with variables, JSON configuration
- **Data Source**: `config.json` (single source of truth for all assignments)
- **Key Pattern**: Config-driven, class-based application architecture

---

## Core Architectural Patterns

### 1. Config-Driven Design
- **Single Source of Truth**: All assignment data lives in `config.json`
- **Never hardcode** assignment titles, descriptions, due dates, or attachment paths
- Use `config.json` as the contract between data and UI
- When adding assignments: update `config.json` first, then verify UI auto-renders

### 2. Class-Based Organization
- **JavaScript Classes**: `AssignmentPortal` (dashboard), `AssignmentPage` (detail view)
- Constructor initializes properties; async `init()` method handles data loading
- Example pattern:
  ```javascript
  class MyComponent {
    constructor(selector) {
      this.element = document.querySelector(selector);
    }
    async init() {
      const data = await fetch('config.json').then(r => r.json());
      this.render(data);
    }
  }
  ```

### 3. Async/Await Pattern
- Use modern `async/await` for all network requests (no callbacks or `.then()` chains)
- Always wrap `fetch()` in try/catch blocks for error handling
- Load config and markdown files via `fetch()`, never inline data

### 4. HTML Generation via String Interpolation
- Build HTML in JavaScript using template literals
- Set via `element.innerHTML` after creating strings
- Keep HTML structure simple; complex styling via CSS variables

### 5. CSS Variables for Theming
- Define colors and spacing in `:root` selector
- Reference via `var(--primary-color)`, `var(--secondary-color)`, etc.
- Makes future theme changes trivial

## Adding & Maintaining Assignments

### Adding a New Assignment

1. **Create assignment folder**:
   ```
   assignments/{assignment-id}/
   ├── README.md          (required)
   ├── starter-code.py    (optional)
   └── data.csv           (optional, for data-analysis tasks)
   ```

2. **Add entry to `config.json`**:
   ```json
   {
     "id": "unique-slug",
     "title": "Assignment Title",
     "description": "One-line summary",
     "path": "assignments/unique-slug",
     "dueDate": "YYYY-MM-DD",
     "attachments": [
       {
         "name": "Starter Code",
         "file": "starter-code.py",
         "type": "python"
       }
     ]
   }
   ```

3. **Create `README.md`** using [templates/assignment-template.md](templates/assignment-template.md):
   - Start with 📘 Assignment title
   - Include 🎯 Objective section
   - List 🛠️ Tasks with numbered requirements
   - End with 🐍 Python and 📁 Files sections if applicable
   - Use emojis consistently for visual hierarchy

4. **Verify**: Dashboard should auto-render and sort by `dueDate`

### Updating Assignment Content
- Edit the assignment's `README.md` directly
- Detail page fetches from disk on every load (no caching issues)
- Changes appear immediately on next page view or refresh

### Due Date Calculation (Automatic)
- **Status logic** compares `dueDate` to current date at 00:00 UTC
- **Urgency levels**:
  - 🔴 **High** (red): 0-3 days remaining, due today, or overdue
  - 🟡 **Medium** (yellow): 4-7 days remaining
  - 🟢 **Low** (green): 8+ days remaining
- Badges auto-update daily; no manual intervention needed

## Code Style & Conventions

### JavaScript
- **Use ES6+ features**: `const`/`let`, arrow functions, template literals, destructuring
- **Async/await required** for all `fetch()` calls
- **Class naming**: PascalCase (`AssignmentPortal`, not `assignmentPortal`)
- **Method naming**: camelCase (`renderAssignments`, not `render_assignments`)
- **No global variables**: Encapsulate logic in classes
- **Comments**: Add JSDoc-style comments to public methods
- **Error handling**: Always wrap fetches in try/catch; log errors to console

### CSS
- **Use CSS variables** for colors, spacing, and font sizes
- **Mobile-first responsive design**: Start with mobile, add `@media` for larger screens
- **BEM naming** (optional but preferred): `.block__element--modifier`
- **Accessibility**: Maintain sufficient color contrast (WCAG AA minimum), test with keyboard nav

### HTML
- **Semantic HTML**: Use `<article>`, `<section>`, `<nav>`, `<header>`, `<footer>` appropriately
- **Accessibility**: Include `alt` text for images, `aria-labels` for buttons, proper heading hierarchy
- **No inline styles**: All styling via CSS classes or variables
- **Generated dynamically**: Avoid static HTML; use JavaScript to render

### File Naming
- Kebab-case for files: `assignment-template.md`, `starter-code.py`
- Underscores in Python: `my_function()`, `my_variable`
- camelCase in JavaScript: `myFunction()`, `myVariable`

## Markdown Content Patterns

### Assignment README Structure
Use this exact structure (see [templates/assignment-template.md](templates/assignment-template.md)):

```
# 📘 Assignment Title

## 🎯 Objective
Clear 1-2 sentence description of what students will learn.

## 🛠️ Tasks
### Task 1: Task Name
- Description of what the student should do
- Specific requirements (numbered sub-items)

### Task 2: Task Name
...

## 🐍 Python Concepts Covered
- Concept 1
- Concept 2

## 📁 Files
- `starter-code.py` — Starter template with function stubs
- Other files as needed
```

### Visual Conventions
- Use emojis at section headers for quick scanning
- Bold key terms: `**list**, **dictionary**, **function`
- Code snippets: ` ```python ` blocks for multi-line code, backticks for inline
- Numbered lists for step-by-step tasks
- Keep descriptions concise and actionable

## When Updating Code

### If Adding UI Features
1. Add relevant data fields to `config.json` schema (if needed)
2. Update the appropriate JavaScript class to fetch and render the data
3. Add styling via CSS variables to `styles.css`
4. Test on mobile and desktop viewports

### If Changing Styling
1. Update CSS variables first (they cascade everywhere)
2. Use `@media` queries for responsive breakpoints
3. Test color contrast with WebAIM Contrast Checker
4. Verify interactive elements remain keyboard-accessible

### If Modifying Assignment Workflow
1. Ensure changes don't break `config.json` schema compatibility
2. Update `README.md` in relevant assignment folders if task instructions change
3. Keep backwards compatibility with existing assignments
